# ShyISA-完整指令集架构定义

## 1. 基础设计

ShyISA 是一个面向教学的汇编语言设计，其设计的哲学是足够简单，友好，帮助初学者入门计算机体系架构，采用阅读友好的大端序。

### 1.1 寄存器架构

系统包含16个**32位**通用寄存器，采用独特的命名方式：0x, 1x, 2x...fx，即16进制数后加字母x。这种命名既简洁又能直观反映寄存器编号。

除通用寄存器外，系统还配备了多个**32位**专用寄存器:

- **程序计数器(PC)**：指向下一条待执行指令
- **段起始寄存器(SEGS)**：当前进程用户段映射到的物理起始地址，仅内核态可修改
- **栈指针寄存器(SP)**：管理栈空间，**需由程序员手动初始化**
- **定时器寄存器(TM)**：定时值寄存器，用于产生周期性 trap
- **状态寄存器(STATUS)**：记录当前运行模式与中断开关
- **Trap入口寄存器(TRAP)**：记录统一 trap 处理程序入口地址
- **异常返回寄存器(EPC)**：记录进入 trap 前的程序计数器
- **异常原因寄存器(CAUSE)**：记录进入 trap 的原因
- **内核栈指针寄存器(KSP)**：保存内核栈指针，用于用户态 trap 进入内核时自动切换栈
- **段结束寄存器(SEGE)**：当前进程用户段的物理结束地址，仅内核态可修改
- **判断结果寄存器(RS)**：存储比较和条件判断的结果
- **退出寄存器(EXIT)**：任何改变都会触发程序退出

### 1.2 特权与 Trap 系统

ShyISA采用最小特权模型，系统只有两种运行模式：

- **内核态(kernel mode)**：可访问全部地址空间和所有特殊寄存器。
- **用户态(user mode)**：可使用通用寄存器和段映射后的用户普通内存，不能直接访问I/O区和受保护特殊寄存器。

当前运行模式与中断开关由 **STATUS寄存器** 保存：

- **bit0**：运行模式，`0` 表示内核态，`1` 表示用户态。
- **bit1**：中断使能，`0` 表示关闭中断，`1` 表示允许中断。
- 其他位保留，写入时应保持为 `0`。

所有异常、中断和系统调用都统一通过 **TRAP/EPC/CAUSE/STATUS** 处理：

1. CPU将旧 **STATUS** 保存到内部 trap 状态栈。该状态栈不映射到地址空间，程序不能直接读写，最小实现深度为64项。超过实现深度继续进入嵌套 trap 属于未定义行为。
2. CPU将进入 trap 前的PC保存到 **EPC**。`syscall` 与 `wait` 唤醒是例外：它们先将PC推进到下一条指令，即 `PC = PC + 12`，再将推进后的PC保存到 **EPC**。因此非法指令、非法地址、权限错误和定时器中断等普通 trap 的 **EPC** 均为进入 trap 前的PC。
3. CPU将进入 trap 的原因写入 **CAUSE**。
4. 若进入 trap 前处于用户态，CPU交换 **SP** 与 **KSP**，使 trap 处理程序从内核栈开始运行。
5. CPU将 `STATUS.bit0` 设置为 `0`，切换到内核态，并将 `STATUS.bit1` 设置为 `0`，关闭可屏蔽中断。
6. CPU将PC设置为 **TRAP** 中保存的地址。

Trap处理完成后，内核应使用 `iret` 指令返回。`iret` 将PC恢复为 **EPC**，并从内部 trap 状态栈恢复旧 **STATUS**。如果恢复后的旧 **STATUS** 表示用户态，CPU交换 **SP** 与 **KSP**，恢复用户栈并保存当前内核栈指针。因此 `iret` 会回到进入 trap 前的运行模式、中断使能状态和对应栈。trap 状态栈为空时执行 `iret` 属于未定义行为。

内部 trap 状态栈只保存 **STATUS**。如果内核允许嵌套 trap，必须在重新开启中断前自行保存 **EPC** 与 **CAUSE**，否则新的 trap 会覆盖原有 **EPC** 与 **CAUSE**。内核态进入嵌套 trap 时不会自动交换 **SP** 与 **KSP**。

最小CAUSE值定义如下：

- `0`：无原因
- `1`：系统调用
- `2`：定时器中断
- `3`：非法指令
- `4`：非法地址
- `5`：权限错误

定时器由 **TM寄存器** 控制。TM默认值为 `0`，表示关闭定时器；非 `0` 时，每10毫秒自动减1。当TM减到 `1` 时，CPU将TM清零并产生一次定时器中断请求。

定时器中断是可屏蔽中断，只在 `STATUS.bit1 = 1` 时进入trap。若定时器到期时中断关闭，CPU保留一个内部 pending 标记；之后当软件重新开启中断时，CPU应在下一条普通指令执行前交付该定时器中断，按统一trap流程设置 `CAUSE = 2` 并跳转到TRAP。这里的重新开启中断包括直接写入 **STATUS**，也包括 `iret` 恢复出 `STATUS.bit1 = 1`；如果 `iret` 后存在 pending 定时器，CPU应在恢复后PC指向的指令执行前先交付定时器trap。

TM是单次定时器。OS若需要周期性时钟、sleep队列或当前时间，应在定时器trap中维护软件计数，并按下一次到期时间重新写入TM。原有的专用定时器入口和返回地址寄存器不再使用。

内核可使用 `wait` 指令暂停CPU，直到出现可交付的 trap 或中断请求。`wait` 仅在内核态且 `STATUS.bit1 = 1` 时有效；若中断关闭或在用户态执行 `wait`，应触发非法指令trap，即 `CAUSE = 3`。若执行 `wait` 时已经存在 pending 且可交付的中断，CPU不暂停，立即按统一trap流程进入trap。`wait` 被中断唤醒时，CPU先将PC推进到下一条指令，即 `PC = PC + 12`，再按统一trap流程保存推进后的PC到 **EPC**；因此处理程序执行 `iret` 后会返回到 `wait` 后面的指令。

内核可使用 `enteruser <va>` 指令首次进入用户态。`enteruser` 仅在内核态可执行；若在用户态执行，应触发非法指令trap，即 `CAUSE = 3`。该指令是原子操作，不走 trap 流程，不压入内部 trap 状态栈。其语义为：先交换 **SP** 与 **KSP**，再将 **STATUS** 设置为 `0b11`（用户态且中断使能），然后将 **PC** 设置为参数给出的用户入口虚拟地址 `va`，最后清空指令缓存，效果等价于执行一次 `fencei`。OS执行 `enteruser` 前应把用户栈指针放入 **KSP**，并让 **SP** 保存当前内核栈指针；执行后 **SP** 为用户栈，**KSP** 为内核栈，供下一次用户态 trap 自动切回内核栈。

### 1.3 内存架构

ShyISA采用**32位字节寻址**系统，每个普通内存地址对应一个8位字节。所有通用寄存器与算术运算仍为32位，32位内存访问按大端序读写连续4个字节。

普通32位内存访问要求地址按4字节对齐；地址不对齐时触发非法地址trap，即 `CAUSE = 4`，CPU不会自动忽略地址低2位。只有 `get8/get16/put8/put16` 这组窄访存指令负责处理字节与半字访问，其中 `get16/put16` 允许非对齐地址。

ShyISA 采用最小段式虚拟内存机制。段转换只影响用户态普通内存访问和用户态取指；内核态始终保持全局物理视角，普通内存虚拟地址等于物理地址，不受 **SEGS/SEGE** 约束。用户态访问普通内存地址 `vaddr >= 0x00000100` 时，转换为物理地址 `paddr = SEGS + vaddr`。若段转换后的本次访问覆盖物理范围超过 `SEGE`，或超出实现内存大小，应触发非法地址trap，即 `CAUSE = 4`。用户态访问I/O区或受保护特殊寄存器时触发权限错误trap，即 `CAUSE = 5`。

CPU复位时 **SEGS** 默认为 `0`，**SEGE** 默认为实现内存末尾，因此未启用进程隔离时用户普通内存与物理内存保持一一对应。OS切换当前进程或进入用户态前应更新 **SEGS/SEGE**。

原子访存指令只作用于普通32位内存地址，目标地址必须4字节对齐，并按普通内存访问规则进行越界、对齐和权限检查。用户态只能对段映射后的普通内存执行原子操作。对特殊映射区、I/O区、保留地址、未对齐地址或越界地址执行原子操作，应触发对应的非法地址或权限错误trap，且不产生部分写入。

原子访存指令的读-改-写作为一个不可分割的操作发生。trap和中断只能在该原子操作之前或之后被观察到，不能观察到其中间状态。多CPU或多hart实现中，针对同一普通内存地址的原子操作必须有单一全局顺序。

### 1.4 I/O子系统

#### 声音系统

四个声音寄存器(M1-M4)分别产生不同波形：

- M1：正弦波
- M2：方波
- M3：三角波
- M4：锯齿波

每个寄存器独立工作，值为0时静音，非0值表示播放频率。

#### UART输入输出

UART通过内存映射方式访问，用于实现字符输入输出等基础I/O。`in/out` 系列指令是硬件或模拟器提供的轻便宿主I/O接口，不要求走UART映射。

### 1.5 指令集设计理念

ShyISA的指令集遵循统一的设计原则：

- 固定长度编码：每条指令占**3个32位字**，即12字节(指令码+参数1+参数2)
- 双操作数架构：绝大多数指令采用双参数格式
- 灵活的寻址模式：通过指令后缀区分地址(a)和立即数(n)操作

核心设计哲学是"一切皆地址"——指令、寄存器、内存、I/O设备都通过统一的地址空间访问，这种设计极大简化了系统架构的理解和实现。

## 2. 地址设计

ShyISA采用统一的地址空间设计，将所有资源映射到**32位**地址范围内。

### 2.1 寄存器地址映射

通用寄存器占据地址空间的最低16个位置：

- **0x00-0x0F**：16个通用寄存器(0x-Fx)

特殊寄存器紧随其后：

- **0x10**：程序计数器(PC)
- **0x11**：段起始寄存器(SEGS)
- **0x12**：栈指针寄存器(SP)
- **0x13**：定时器寄存器(TM)
- **0x14**：状态寄存器(STATUS)
- **0x15**：Trap入口寄存器(TRAP)
- **0x16-0x19**：声音寄存器(M1-M4)
- **0x1A**：结果寄存器(RS)
- **0x1B**：退出寄存器(EXIT)
- **0x1C**：异常返回寄存器(EPC)
- **0x1D**：异常原因寄存器(CAUSE)
- **0x1E**：内核栈指针寄存器(KSP)
- **0x1F**：段结束寄存器(SEGE)

### 2.2 指令空间

指令操作码占据0x20-0x6F地址范围，为未来扩展预留了充足空间。

目前，0x62-0x6F为保留空间。取指遇到保留操作码时触发非法指令trap，即 `CAUSE = 3`。

### 2.3 I/O设备映射

UART通过内存映射实现：

- **0x70**：UART数据寄存器。读取得到一个输入字节；写入发送低8位作为输出字节。
- **0x71**：UART状态寄存器。bit0表示有可读输入，bit1表示可写输出，其他位保留为0。
- **0x72-0x7F**：保留用于扩展UART控制寄存器。
- **0x80-0xFF**：保留用于扩展I/O设备。

### 2.4 内存布局

系统从 **0x00000100** 开始的内存空间可用于普通内存访问，提供了灵活的内存管理方案。代码段长度可精确预测，因此用户可根据需求自定义内存布局，甚至实现代码自修改。自修改代码写入目标指令后，必须执行 `fencei`，才能保证后续取指观察到更新后的指令内容。

内核态不启用段转换，普通内存地址就是全局物理地址。用户态启用段转换，普通内存地址按 `SEGS + vaddr` 转换为物理地址，访问末尾不能超过 `SEGE`。

用户态访问I/O区或受保护特殊寄存器时，应触发权限错误trap，即 `CAUSE = 5`。用户态通过写 **PC**、跳转或返回等方式执行普通内存地址时，同样按用户段转换取指。

用户态可直接访问以下特殊寄存器：**PC**、**SP**、**M1-M4**、**RS**、**EXIT**。其他特殊寄存器，包括 **SEGS**、**TM**、**STATUS**、**TRAP**、**EPC**、**CAUSE**、**KSP**、**SEGE**，仅内核态可直接访问。

`0x00000000-0x000000FF` 是寄存器、指令操作码和 I/O 的特殊映射区，不作为普通内存处理，也不套用普通32位内存访问的4字节对齐约束。普通内存从 `0x00000100` 开始。保留地址不是可用存储单元；对 `0x62-0x6F`、`0x72-0xFF` 等保留地址进行普通读写时，应触发非法地址trap，即 `CAUSE = 4`。`0x20-0x6F` 仅作为指令操作码取值使用，不作为可读写的数据存储单元。

系统有以下硬性约束：

1. **程序员必须在程序开始时手动初始化栈指针(SP)**，例如 `setn sp 0xEFFFFFFC`。SP使用字节地址并保持4字节对齐，指向下一次push要写入的位置。
2. 程序从 **0x00000100** 地址开始执行。
3. CPU启动时处于内核态。
4. OS在进入用户态前必须初始化 **KSP**，并保持4字节对齐，使用户态 trap 能自动切换到有效内核栈。
5. OS在切换进程或进入用户态前应设置 **SEGS/SEGE**，确保用户普通内存映射到当前进程允许访问的物理内存段。

## 3. 算术运算约定

ShyISA的算术运算遵循简单而一致的规则：

### 3.1 数值表示

所有算术运算均采用无符号数表示。

### 3.2 溢出处理

运算结果超出**32位**范围时，系统自动截断高位，保留低**32位**结果。

### 3.3 除法特殊情况

除以零操作不会引发异常，而是返回最大值 **0xFFFFFFFF** 作为结果。

### 3.4 条件判断机制

所有比较指令通过RS寄存器反馈结果：条件成立时RS置1，不成立时置0。

### 3.5 位移特殊情况

左移或右移的位移量大于等于32时，结果为 `0`。

## 4. 软件约定

### 4.1 程序加载

系统通过串口或其他通信方式接收链接器生成的 `.sfs` raw 内存镜像。镜像中的字节按文件偏移直接映射到同值地址；模拟器不解析 section、symbol 或 relocation，这些信息只存在于 `.sobj` object 文件和链接阶段。

### 4.2 指令编码格式

每条指令遵循统一的三字格式，总长度为12字节：

- **第一个字**：指令操作码地址
- **第二个字**：参数1
- **第三个字**：参数2

单参数或无参数指令自动在未使用的参数位置填充0，保证所有指令长度一致。

除跳转、调用、返回和 trap 类指令外，普通指令执行完成后 `PC = PC + 12`。

每条已开始执行的指令在架构可见状态上以完整指令为边界提交。可屏蔽中断不会在一条指令的中间交付；同步异常若在执行过程中产生，该指令除规范明确说明的写入外不产生部分状态更新。

32位内存读写指令按大端序访问连续4个字节，并要求普通内存地址4字节对齐。不对齐时触发非法地址trap，即 `CAUSE = 4`。`get8/get16/put8/put16` 是最小窄访存口子，负责字节和半字访问，允许非对齐地址。

指令取指、32位普通访存和栈操作均不自动清零地址低2位。若访问普通内存时地址不满足对应对齐要求，应触发非法地址trap。特殊映射区地址不受普通内存对齐约束。

访问超出实现内存大小的普通内存地址时，应触发非法地址trap，即 `CAUSE = 4`。

对于规范中标记为未定义行为的致命状态，例如未初始化 **TRAP** 时发生 trap、trap 状态栈溢出或空栈 `iret`，模拟器实现应直接 panic 并停止执行，便于定位系统错误。

### 4.3 二进制文件格式

最终生成的可执行文件采用 `.sfs` 扩展名。`.sfs` 是链接器把 `.sobj` 中的 section 布局、重定位并写入目标地址后得到的 raw 内存镜像，不是带有文件头的 object 格式，也不保存 section 表、symbol 表或 relocation 表。

ShyISA 的地址是32位字节地址，`.sfs` 文件是字节流。地址 `A` 对应文件中的字节偏移 `A`。每个32位字使用大端序存储。例如地址 `A` 开始的连续4个字节 `0x80 0x81 0x82 0x83` 表示一个32位字 `0x80818283`。

`.sfs` 的内容由链接器按 `ObjFormat.md` 中定义的 section 规则生成：

- `text._start` 必须存在，并放到程序入口地址 `0x00000100`。
- 其他 `text.*` section 接在 `text._start` 后面。
- `data` 和 `data.*` section 接在所有 `text.*` section 后面。
- 其他 section 名暂不定义默认布局，链接器可以报错。
- section 起始地址按4字节对齐。
- 未被任何 section 写入但位于文件长度内的 gap 应填充为 `0x00`。

`.sfs` 的低地址保留两个32位 metadata word，供OS在加载程序前读取：

- 文件偏移 `0x00000004`，即寄存器映射中 `1x` 对应 word：程序内存提示，单位字节。
- 文件偏移 `0x00000008`，即寄存器映射中 `2x` 对应 word：程序栈提示，单位字节。

链接器从所有输入 `.sobj` 汇总这些提示。多个 object 的同类提示相加；若所有 object 都没有声明内存提示，则写入默认 `32M`；若所有 object 都没有声明栈提示，则写入默认 `4K`。模拟器不解释这些 metadata，OS可按约定读取并决定如何分配进程空间。

`.sfs` 的文件长度应至少覆盖最高已写入 section 字节的后一字节；若没有任何 section 写入超过入口前保留区，也不应小于 `0x00000100` 字节。模拟器可以把文件末尾之后到实现内存大小之间的普通内存视为 `0x00` 初始化，也可以要求镜像扩展到固定内存大小。访问超出实现内存大小的地址仍按非法地址处理。

### 4.4 数据布局

普通内存的最小可寻址单元是8位字节。汇编中的单字符用于字节数据时写入一个字节，字符串按 UTF-8 编码后的每个字节连续写入，并在末尾写入一个值为 `0` 的字节。32位立即数和指令字按大端序写入连续4个字节。需要输出单个 Unicode 字符时，应将 Unicode code point 作为32位值存放或作为立即数传给 `oututfa`/`oututfn`。

## 5. 汇编格式

### 5.1 程序结构

ShyISA 汇编源码仍使用 `___DEFINE___`、`___DATA___`、`___CODE___` 三个顶层区域。顶层区域只用于组织源码，不直接等同于最终二进制布局。最终二进制布局由链接器根据 object 文件中的 section 决定。

- `___DEFINE___`：定义当前源码文件内可用的常量别名。每行格式为 `<name> <value>`，其中 `<value>` 只能是立即数（十进制、`0x`/`0X` 十六进制、`b` 后缀二进制）。寄存器名（如 `sp`、`1x`）有固定含义，不需要在 DEFINE 中重定义，也不允许 DEFINE 的名字与寄存器名冲突。DEFINE 中的名字和值均大小写不敏感。
- `___DATA___`：书写数据内容。若没有显式 `.section`，默认写入 `data` section。
- `___CODE___`：书写指令内容。若没有显式 `.section`，默认写入 `text._start` section。

默认 section 主要用于兼容简单单文件程序。需要多文件链接时，除入口文件的 `text._start` 外，源码应显式使用不会与其他 object 冲突的 section 名。

汇编器不直接决定 section 的最终内存地址。汇编器只生成 object 文件，记录每个 section 的内容、对外符号和需要链接器补地址的位置。链接器再把多个 object 文件合并为最终 `.sfs` 内存镜像。

### 5.2 Section 与符号

ShyISA 汇编支持以下最小链接语义。

#### `.section <name>`

切换当前输出 section。section 是链接器放置代码和数据的最小单位。

```
.section text._start
```

section 名本身也是一个对外可见符号，表示该 section 的起始地址。链接完成前，section 符号只表示“该 section 的开头”。链接完成后，它才拥有最终绝对地址。

`text._start` 是约定的入口 section。默认链接规则必须把 `text._start` 放到程序入口地址 `0x00000100`。其他 `text.*` section 接在 `text._start` 后面，`data` 和 `data.*` section 接在所有 `text.*` section 后面，section 起始地址按4字节对齐。链接脚本属于后续扩展，不在本版规范中定义；完整 object 和 section 规则以 `ObjFormat.md` 为准。

同一个 object 文件内重复切换到同名 section 表示继续向该 section 追加内容。链接同一批 object 文件时，不允许出现重名 section，因为 section 名会作为对外符号暴露。

#### `.symbol <name>`

在当前 section 的当前位置定义一个对外可见符号。

```
.section text.print
.symbol print
    oututfa 1x
    ret
```

`.symbol` 不写入任何字节，只记录名字与当前 section 内偏移。该符号可以被其他 object 文件引用，也可以作为 `jmpn`、`calln`、`setn` 等指令的参数。链接完成后，链接器把它解析为最终绝对地址。

#### `<label>:`

定义当前 section 内的局部 label。

```
.section text.main
loop:
    addn 1x 1
    sman 1x 10
    jmpn loop
```

局部 label 不对外暴露，只能在当前源码文件内使用。汇编器在编译当前文件时必须能解析局部 label 属于哪个 section 以及它在 section 内的偏移。

如果局部 label 被写入指令或数据中的 32 位地址字段，汇编器仍然不能写入最终绝对地址，因为 section 还没有被链接到最终位置。此时汇编器应写入占位值，并在 object 文件中记录一次重定位。链接器根据 section 最终地址和 label 偏移补齐该字段。

### 5.3 指令与地址引用

指令格式仍为固定 3 个 32 位字。汇编源码中的指令参数可以引用：

- 数字立即数，例如 `123`、`0x100`、`1100b`
- 寄存器名或别名，例如 `1x`、`sp`
- 当前文件内的局部 label
- 当前文件或其他文件导出的 `.symbol`
- section 名
- 链接期地址偏移表达式，例如 `message(4)`、`loop(12)`、`data.table(8)`

ShyISA 汇编不需要 `.extern`。如果一个名字被引用，但既不是数字、寄存器、`___DEFINE___` 名字、当前文件局部 label，也不是当前文件定义的 `.symbol` 或 section 名，汇编器应把它作为外部符号引用写入 relocation，交给链接器解析。

链接期地址偏移表达式的格式为 `<base>(<offset>)`。`base` 必须是 section 名、当前文件内的局部 label、当前文件或外部 object 导出的 symbol；`offset` 是无符号立即数，支持十进制、`0x`/`0X` 十六进制和 `b` 后缀二进制。表达式的值为 `base` 最终绝对地址加上 `offset`，由链接器在处理 relocation 时补齐。该语法只描述链接期地址计算，不表示寄存器间接寻址；`ax(8)` 这类“寄存器值加偏移”的运行期地址模式不属于本语法。

例如：

```
.section data.message
.symbol message
"Hello!"

.section text.print
.symbol print
    ret

.section text._start
.symbol _start
loop:
calln print
setn 1x message
setn 2x message(4)
jmpn loop
jmpn loop(12)
```

其中 `print`、`message`、`message(4)`、`loop`、`loop(12)` 的最终地址是否已知，取决于它们是不是在当前 section 内以及当前 section 是否已经完成链接。统一规则是：只要写入的是最终绝对地址字段，汇编器就可以在 object 文件中留下重定位记录，由链接器最终填值。

### 5.4 数据语法

数据内容可以写在任意 section 中。推荐把可执行指令放入 `text.*` section，把初始化数据放入 `data.*` section。

数据初始化行的格式为 `<address-expr> <value>`。`address-expr` 表示写入位置，可以是绝对地址、section 名、symbol、局部 label，或链接期地址偏移表达式，例如 `message(0)`、`message(4)`、`data.table(8)`。`value` 支持以下基础写法：

- 字符串：`"Hello!"`，按 UTF-8 字节写入，并在末尾写入 `0`
- 32 位数：`123`、`0x1234`、`1100b`，按大端序写入 4 字节
- 符号引用：`message`，作为 32 位地址字段写入，由链接器补齐
- 链接期地址偏移表达式：`message(4)`、`data.table(8)`，作为 32 位地址字段写入，由链接器补齐
- 数组：`{111, 222, 114514}`，每个元素按 32 位大端序写入
- 字节数组：`[104, 101, 108, 108, 111]`，每个元素按 8 位字节写入，元素值不能超过 `255`

旧式绝对地址数据初始化仍可作为兼容语法；这类写法不属于推荐的可链接示例：

```
0x00002000 "Hello!"
```

但可链接源码应优先使用 section 和符号作为写入位置：

```
.section data.message
.symbol message
message "Hello!"
message(6) 0
```

### 5.5 汇编示例

下面是一个完整的ShyISA汇编程序示例，展示了各部分的使用方法：

```

___DEFINE___

PI 314159          // 定义常量PI（十进制）

STACK_INIT 0xEFFFFFFC  // 栈初始化地址（十六进制）

FLAG 1100b         // 标志位常量（二进制）



___DATA___

.section data.message
.symbol message
message "Hello!"

.section data.counter
.symbol counter
counter 12345678

.section data.ptrs
.symbol message_ptrs
message_ptrs message(1) // 存储 message 最终地址+1，由链接器补齐


___CODE___

.section text._start
.symbol _start
setn sp STACK_INIT  // 手动初始化栈指针，保持4字节对齐

setn 1x 1          // 将寄存器1x设置为1

loop:              // 定义当前section内的局部label

addn counter 1     // counter由链接器解析为最终地址

oututfa message    // message由链接器解析为最终地址

oututfa message(1) // message最终地址+1，由链接器解析

outn PI            // 以十六进制输出宏定义的常量

addn 1x 1          // 寄存器1x加1

sman 1x 10         // 比较1x是否小于10

jmpn loop          // 如果条件成立，跳转到局部label

ujmpn loop(12)     // 无条件跳转到loop后面一条指令

```

这个示例展示了宏定义、section、对外符号、局部 label、链接期地址偏移表达式、数据初始化、基本运算、I/O 操作和条件跳转等核心功能的使用。
