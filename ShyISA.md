# ShyISA-完整指令集架构定义

## 1. 基础设计

ShyISA 是一个面向教学的汇编语言设计，其设计的哲学是足够简单，友好，帮助初学者入门计算机体系架构，采用阅读友好的大端序。

### 1.1 寄存器架构

系统包含16个**32位**通用寄存器，采用独特的命名方式：0x, 1x, 2x...fx，即16进制数后加字母x。这种命名既简洁又能直观反映寄存器编号。

除通用寄存器外，系统还配备了多个**32位**专用寄存器:

- **程序计数器(PC)**：指向下一条待执行指令
- **栈指针寄存器(SP)**：管理栈空间，**需由程序员手动初始化**
- **定时器寄存器(TM)**：定时值寄存器，用于产生周期性 trap
- **状态寄存器(STATUS)**：记录当前运行模式与中断开关
- **Trap入口寄存器(TRAP)**：记录统一 trap 处理程序入口地址
- **异常返回寄存器(EPC)**：记录进入 trap 前的程序计数器
- **异常原因寄存器(CAUSE)**：记录进入 trap 的原因
- **内核栈指针寄存器(KSP)**：保存内核栈指针，用于用户态 trap 进入内核时自动切换栈
- **判断结果寄存器(RS)**：存储比较和条件判断的结果
- **退出寄存器(EXIT)**：任何改变都会触发程序退出

### 1.2 特权与 Trap 系统

ShyISA采用最小特权模型，系统只有两种运行模式：

- **内核态(kernel mode)**：可访问全部地址空间和所有特殊寄存器。
- **用户态(user mode)**：可使用通用寄存器和用户内存区，不能直接访问内核区、I/O区和受保护特殊寄存器。

当前运行模式与中断开关由 **STATUS寄存器** 保存：

- **bit0**：运行模式，`0` 表示内核态，`1` 表示用户态。
- **bit1**：中断使能，`0` 表示关闭中断，`1` 表示允许中断。
- 其他位保留，写入时应保持为 `0`。

所有异常、中断和系统调用都统一通过 **TRAP/EPC/CAUSE/STATUS** 处理：

1. CPU将旧 **STATUS** 保存到内部 trap 状态栈。该状态栈不映射到地址空间，程序不能直接读写，最小实现深度为64项。超过实现深度继续进入嵌套 trap 属于未定义行为。
2. CPU将进入 trap 前的PC保存到 **EPC**。`syscall` 与 `wait` 唤醒是例外：它们先将PC推进到下一条指令，即 `PC = PC + 12`，再将推进后的PC保存到 **EPC**。因此非法指令、非法地址、权限错误和定时器中断等普通 trap 的 **EPC** 均为进入 trap 前的PC。
3. CPU将进入 trap 的原因写入 **CAUSE**。
4. 若进入 trap 前处于用户态，CPU交换 **SP** 与 **KSP**，使 trap 处理程序从内核栈开始运行。
5. CPU将 `STATUS.bit0` 设置为 `0`，切换到内核态，并将 `STATUS.bit1` 设置为 `0`，关闭可屏蔽中断。
6. CPU将PC设置为 **TRAP** 中保存的地址。

Trap处理完成后，内核应使用 `iret` 指令返回。`iret` 将PC恢复为 **EPC**，并从内部 trap 状态栈恢复旧 **STATUS**。如果恢复后的旧 **STATUS** 表示用户态，CPU交换 **SP** 与 **KSP**，恢复用户栈并保存当前内核栈指针。因此 `iret` 会回到进入 trap 前的运行模式、中断使能状态和对应栈。trap 状态栈为空时执行 `iret` 属于未定义行为。

内部 trap 状态栈只保存 **STATUS**。如果内核允许嵌套 trap，必须在重新开启中断前自行保存 **EPC** 与 **CAUSE**，否则新的 trap 会覆盖原有 **EPC** 与 **CAUSE**。内核态进入嵌套 trap 时不会自动交换 **SP** 与 **KSP**。

最小CAUSE值定义如下：

- `0`：无原因
- `1`：系统调用
- `2`：定时器中断
- `3`：非法指令
- `4`：非法地址
- `5`：权限错误

定时器由 **TM寄存器** 控制。TM默认值为 `0`，表示关闭定时器；非 `0` 时，每10毫秒自动减1。当TM减到 `1` 时，CPU将TM清零并产生一次定时器中断请求。

定时器中断是可屏蔽中断，只在 `STATUS.bit1 = 1` 时进入trap。若定时器到期时中断关闭，CPU保留一个内部 pending 标记；之后当软件重新开启中断时，CPU应在下一条普通指令执行前交付该定时器中断，按统一trap流程设置 `CAUSE = 2` 并跳转到TRAP。这里的重新开启中断包括直接写入 **STATUS**，也包括 `iret` 恢复出 `STATUS.bit1 = 1`；如果 `iret` 后存在 pending 定时器，CPU应在恢复后PC指向的指令执行前先交付定时器trap。

TM是单次定时器。OS若需要周期性时钟、sleep队列或当前时间，应在定时器trap中维护软件计数，并按下一次到期时间重新写入TM。原有的专用定时器入口和返回地址寄存器不再使用。

内核可使用 `wait` 指令暂停CPU，直到出现可交付的 trap 或中断请求。`wait` 仅在内核态且 `STATUS.bit1 = 1` 时有效；若中断关闭或在用户态执行 `wait`，应触发非法指令trap，即 `CAUSE = 3`。若执行 `wait` 时已经存在 pending 且可交付的中断，CPU不暂停，立即按统一trap流程进入trap。`wait` 被中断唤醒时，CPU先将PC推进到下一条指令，即 `PC = PC + 12`，再按统一trap流程保存推进后的PC到 **EPC**；因此处理程序执行 `iret` 后会返回到 `wait` 后面的指令。

### 1.3 内存架构

ShyISA采用**32位字节寻址**系统，每个普通内存地址对应一个8位字节。所有通用寄存器与算术运算仍为32位，32位内存访问按大端序读写连续4个字节。

普通32位内存访问要求地址按4字节对齐；地址不对齐时触发非法地址trap，即 `CAUSE = 4`，CPU不会自动忽略地址低2位。只有 `get8/get16/put8/put16` 这组窄访存指令负责处理字节与半字访问，其中 `get16/put16` 允许非对齐地址。

### 1.4 I/O子系统

#### 声音系统

四个声音寄存器(M1-M4)分别产生不同波形：

- M1：正弦波
- M2：方波
- M3：三角波
- M4：锯齿波

每个寄存器独立工作，值为0时静音，非0值表示播放频率。

#### UART输入输出

UART通过内存映射方式访问，用于实现字符输入输出等基础I/O。`in/out` 系列指令是硬件或模拟器提供的轻便宿主I/O接口，不要求走UART映射。

### 1.5 指令集设计理念

ShyISA的指令集遵循统一的设计原则：

- 固定长度编码：每条指令占**3个32位字**，即12字节(指令码+参数1+参数2)
- 双操作数架构：绝大多数指令采用双参数格式
- 灵活的寻址模式：通过指令后缀区分地址(a)和立即数(n)操作

核心设计哲学是"一切皆地址"——指令、寄存器、内存、I/O设备都通过统一的地址空间访问，这种设计极大简化了系统架构的理解和实现。

## 2. 地址设计

ShyISA采用统一的地址空间设计，将所有资源映射到**32位**地址范围内。

### 2.1 寄存器地址映射

通用寄存器占据地址空间的最低16个位置：

- **0x00-0x0F**：16个通用寄存器(0x-Fx)

特殊寄存器紧随其后：

- **0x10**：程序计数器(PC)
- **0x11**：保留地址
- **0x12**：栈指针寄存器(SP)
- **0x13**：定时器寄存器(TM)
- **0x14**：状态寄存器(STATUS)
- **0x15**：Trap入口寄存器(TRAP)
- **0x16-0x19**：声音寄存器(M1-M4)
- **0x1A**：结果寄存器(RS)
- **0x1B**：退出寄存器(EXIT)
- **0x1C**：异常返回寄存器(EPC)
- **0x1D**：异常原因寄存器(CAUSE)
- **0x1E**：内核栈指针寄存器(KSP)
- **0x1F**：保留地址

### 2.2 指令空间

指令操作码占据0x20-0x6F地址范围，为未来扩展预留了充足空间。

目前，0x5F-0x6F为保留空间。取指遇到保留操作码时触发非法指令trap，即 `CAUSE = 3`。

### 2.3 I/O设备映射

UART通过内存映射实现：

- **0x70**：UART数据寄存器。读取得到一个输入字节；写入发送低8位作为输出字节。
- **0x71**：UART状态寄存器。bit0表示有可读输入，bit1表示可写输出，其他位保留为0。
- **0x72-0x7F**：保留用于扩展UART控制寄存器。
- **0x80-0xFF**：保留用于扩展I/O设备。

### 2.4 内存布局

系统从 **0x00000100** 开始的内存空间完全可读写，提供了灵活的内存管理方案。代码段长度可精确预测，因此用户可根据需求自定义内存布局，甚至实现代码自修改。

为支持最小OS，ShyISA采用固定内核/用户内存划分：

- **0x00000100-0x000FFFFF**：内核区，仅内核态可读写执行。
- **0x00100000及之后**：用户区，用户态可读写执行。

用户态访问内核区、I/O区或受保护特殊寄存器时，应触发权限错误trap，即 `CAUSE = 5`。用户态通过写 **PC**、跳转或返回等方式试图执行内核区地址时，同样触发权限错误trap。

用户态可直接访问以下特殊寄存器：**PC**、**SP**、**M1-M4**、**RS**、**EXIT**。其他特殊寄存器，包括 **TM**、**STATUS**、**TRAP**、**EPC**、**CAUSE**、**KSP**，仅内核态可直接访问。

`0x00000000-0x000000FF` 是寄存器、指令操作码和 I/O 的特殊映射区，不作为普通内存处理，也不套用普通32位内存访问的4字节对齐约束。普通内存从 `0x00000100` 开始。保留地址不是可用存储单元；对 `0x11`、`0x1F`、`0x5F-0x6F`、`0x72-0xFF` 等保留地址进行普通读写时，应触发非法地址trap，即 `CAUSE = 4`。`0x20-0x6F` 仅作为指令操作码取值使用，不作为可读写的数据存储单元。

系统有以下硬性约束：

1. **程序员必须在程序开始时手动初始化栈指针(SP)**，例如 `setn sp 0xEFFFFFFC`。SP使用字节地址并保持4字节对齐，指向下一次push要写入的位置。
2. 程序从 **0x00000100** 地址开始执行。
3. CPU启动时处于内核态。
4. OS在进入用户态前必须初始化 **KSP**，并保持4字节对齐，使用户态 trap 能自动切换到有效内核栈。

## 3. 算术运算约定

ShyISA的算术运算遵循简单而一致的规则：

### 3.1 数值表示

所有算术运算均采用无符号数表示。

### 3.2 溢出处理

运算结果超出**32位**范围时，系统自动截断高位，保留低**32位**结果。

### 3.3 除法特殊情况

除以零操作不会引发异常，而是返回最大值 **0xFFFFFFFF** 作为结果。

### 3.4 条件判断机制

所有比较指令通过RS寄存器反馈结果：条件成立时RS置1，不成立时置0。

### 3.5 位移特殊情况

左移或右移的位移量大于等于32时，结果为 `0`。

## 4. 软件约定

### 4.1 程序加载

系统通过串口或其他通信方式接收完整的内存镜像文件。镜像直接映射到内存地址空间，无需额外的加载过程。

### 4.2 指令编码格式

每条指令遵循统一的三字格式，总长度为12字节：

- **第一个字**：指令操作码地址
- **第二个字**：参数1
- **第三个字**：参数2

单参数或无参数指令自动在未使用的参数位置填充0，保证所有指令长度一致。

除跳转、调用、返回和 trap 类指令外，普通指令执行完成后 `PC = PC + 12`。

32位内存读写指令按大端序访问连续4个字节，并要求普通内存地址4字节对齐。不对齐时触发非法地址trap，即 `CAUSE = 4`。`get8/get16/put8/put16` 是最小窄访存口子，负责字节和半字访问，允许非对齐地址。

指令取指、32位普通访存和栈操作均不自动清零地址低2位。若访问普通内存时地址不满足对应对齐要求，应触发非法地址trap。特殊映射区地址不受普通内存对齐约束。

访问超出实现内存大小的普通内存地址时，应触发非法地址trap，即 `CAUSE = 4`。

对于规范中标记为未定义行为的致命状态，例如未初始化 **TRAP** 时发生 trap、trap 状态栈溢出或空栈 `iret`，模拟器实现应直接 panic 并停止执行，便于定位系统错误。

### 4.3 二进制文件格式

最终生成的可执行文件采用 `.sfs` 扩展名(ShyISA File System)，文件格式应当是完整内存映射的 raw 序列化。

ShyISA 的地址是32位字节地址，`.sfs` 文件是字节流。地址 `A` 对应文件中的字节偏移 `A`。每个32位字使用大端序存储。例如地址 `A` 开始的连续4个字节 `0x80 0x81 0x82 0x83` 表示一个32位字 `0x80818283`。

`.sfs` 的大小取决于内存大小，不应小于 `0x00000100` 字节（这是寄存器、指令操作码和 I/O 映射保留空间）。

注：编译器和模拟器可优先实现2MiB、4MiB、8MiB等固定字节大小。

注2：由于程序自行分配 `0x00000100` 后的空间，`.sfs` 的空间应当符合程序要求。如程序使用了字节地址 `0x00200000`，那么 `.sfs` 就不应该小于 `0x00200001` 字节。

### 4.4 数据布局

普通内存的最小可寻址单元是8位字节。汇编中的单字符写入一个字节，字符串按每个字符一个字节连续写入，并在末尾写入一个值为 `0` 的字节。32位立即数和指令字按大端序写入连续4个字节。

## 5. 汇编格式

### 5.1 程序结构

ShyISA汇编程序由三个主要部分组成，每部分都有的程序指令，支持标签定义用于跳转和函数调用。

### 5.2 汇编示例

下面是一个完整的ShyISA汇编程序示例，展示了各部分的使用方法：

```

___DEFINE___

SP sp              // 定义栈指针别名

PI 314159          // 定义常量PI



___DATA___

// 数据初始化支持多种格式

//这里相当于往堆空间初始化全局变量



0x00210000 "Hello!" // 字符串，每个字符占一个字节，并以0结尾

0x00200000 'A'      // 单字符ASCII码

0x00200004 12345678 // 32位立即数，按4字节对齐

0x00200008 {111, 222, 114514}  // 数组初始化，支持32位立即数，以及单字符；32位元素按4字节对齐
/*也支持这种注释*/
/*还支持0x 0X开头的十六进制以及 b结尾的二进制
example：0x114 0X514 1100b */


___CODE___

setn sp 0x00FFFFFC // 手动初始化栈指针，保持4字节对齐

setn 1x 1          // 将寄存器1x设置为1

.start             // 定义标签，对应下一条指令地址(0x00000118)

addn 0x00200004 1  // 地址0x00200004的值加1

outaasc 0x00200000 // 输出ASCII字符

outn PI            // 输出宏定义的常量

addn 1x 1          // 寄存器1x加1

sman 1x 10         // 比较1x是否小于10

jmpn .start        // 如果条件成立，跳转到标签.start

```

这个示例展示了宏定义、数据初始化、基本运算、I/O操作和条件跳转等核心功能的使用。
